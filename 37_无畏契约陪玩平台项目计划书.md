一、项目概述
1. 项目背景
当前《无畏契约》玩家数量持续增长，大量中低段位玩家存在技术提升、组队开黑、游戏陪伴需求，而高段位玩家有技能变现意愿。市面上缺少垂直、安全、透明的无畏契约专属陪玩平台。本项目旨在搭建一个连接玩家与优质陪玩者的信息系统，解决匹配低效、交易不安全、服务不规范等问题，提升用户游戏体验，推动电竞陪玩服务规范化。
2. 项目目标
实现陪玩者与普通用户的智能匹配、订单全流程管理
提供无畏契约游戏数据展示、战绩分析、能力对比功能
构建安全可靠的支付、结算、评价与投诉体系
完成前后端分离、可部署、可演示的完整信息系统
满足课程设计要求，具备完整文档、代码、测试与分工
3. 开发环境与条件
开发工具：IDEA / VS Code
数据库：MySQL 8.0
后端：Java Spring Boot / Python Flask
前端：Vue.js + Element UI + Axios
运行环境：Windows 10/11、Tomcat、本地服务器 / 阿里云学生机
依赖：Websocket、HTTPS、第三方支付接口（模拟）
二、需求分析
1. 目标用户与使用场景
普通玩家：发布陪玩需求、下单、跟踪订单、评价、查看游戏数据
陪玩师：注册认证、技能标签、接单、服务、结算提现
管理员：审核资质、监管订单、处理投诉、运营配置
使用场景：找陪玩上分、娱乐开黑、技术教学、订单交易、信誉评价
2. 核心功能清单
用户注册 / 登录 / 实名认证
陪玩师入驻、审核、技能评级、标签管理
需求发布、智能匹配、手动选择陪玩师
订单创建、支付、状态跟踪、完成评价
无畏契约战绩、KDA、胜率、角色数据展示
支付结算、平台抽成、提现管理
投诉举报、违规处理、信誉评分
后台管理、数据统计、运营配置
3. 非功能需求
性能要求：接口响应 < 1 秒，支持 50 人同时在线
安全要求：密码加密、数据脱敏、防 SQL 注入、HTTPS 传输
兼容性要求：支持 PC 浏览器、H5 页面、移动端访问
可靠性要求：数据定时备份，7×24 稳定运行
三、技术路线
1. 总体架构
采用四层架构：
表现层：Web 端、H5、管理后台
应用层：用户服务、陪玩服务、订单服务、支付服务、数据服务
数据层：MySQL 存储用户、订单、陪玩师、交易、游戏数据
基础设施层：云服务器、CDN、安全防护、负载均衡
2. 主要技术
前端：Vue2 + Element UI + Axios
后端：Spring Boot（Java）
数据库：MySQL
实时通信：WebSocket（订单 / 消息推送）
部署：Tomcat + 阿里云学生机
安全：MD5 加密、验证码、防注入、权限控制
3. 数据设计初步考虑
主要实体：用户、陪玩师、订单、游戏数据、评价、交易记录
关键数据表
user（用户信息）
master（陪玩师信息）
orders（订单信息）
game_data（游戏战绩）
payment（支付结算）
review（评价反馈）
