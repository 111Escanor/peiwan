## 一、项目概述

### 1. 项目背景
《无畏契约》作为一款热门战术射击游戏，玩家数量持续增长。大量中低段位玩家存在技术提升、组队开黑、游戏陪伴等需求，而高段位玩家有技能变现的意愿。然而，市面上缺少垂直、安全、透明的无畏契约专属陪玩平台。本项目旨在搭建一个连接玩家与陪玩者的信息系统，解决匹配低效、交易不安全、服务不规范等问题，提升用户游戏体验，推动电竞陪玩服务规范化。

### 2. 系统目标
- 实现陪玩者与普通用户的智能匹配与订单全流程管理
- 提供无畏契约游戏数据展示、战绩分析、能力对比功能
- 构建安全可靠的支付、结算、评价与投诉体系
- 完成前后端分离、可部署、可演示的完整信息系统
- 满足课程设计要求，具备完整文档、代码、测试与分工证据

## 二、需求分析

### 1. 功能需求
系统面向三类用户：**普通玩家**、**陪玩师**、**管理员**。

| 角色 | 核心功能 |
|------|----------|
| 普通玩家 | 注册/登录、浏览陪玩师、发布陪玩需求、选择陪玩师下单、模拟支付、跟踪订陪玩师 | 注册并申请认证、设置技能标签与价格、接单、开始/结束服务、查看收益记录、申请提现（模拟） |单状态、评价陪玩师、查看个人游戏战绩（模拟数据） |

| 管理员 | 登录后台、审核陪玩师资质、管理用户（禁用/解禁）、处理投诉举报、查看平台运营统计数据 |

### 2. 非功能需求
- **性能要求**：核心接口响应时间 < 1秒，支持至少50人同时在线。
- **安全要求**：密码使用BCrypt加密存储，JWT进行无状态认证，防SQL注入（MyBatis参数化查询），关键操作需权限校验。
- **兼容性要求**：支持主流PC浏览器（Chrome，Edge），页面采用响应式布局，适配移动端访问。
- **可靠性要求**：数据库每日备份，系统关键状态（订单、支付）有事务保证。

## 四、系统实现

### 1. 关键技术
- **智能匹配（简化）**：根据玩家选择的段位要求和价格范围，筛选符合条件的陪玩师并排序。
- **实时推送**：使用WebSocket实现“陪玩师接单提醒”和“订单状态变更通知”。
- **模拟支付**：后端提供 `/api/pay/mock` 接口，不涉及真实资金流转，仅记录交易流水并更新订单状态。
- **权限控制**：基于JWT + Spring Security拦截器，区分普通用户、陪玩师、管理员权限。

### 3. 核心代码片段

**JWT 工具类（生成与验证）**
```java
public class JwtUtil {
    private static final String SECRET = "mySecretKey";
    private static final long EXPIRE = 86400000L; // 24小时

    public static String generateToken(String userId, String role) {
        return Jwts.builder()
            .setSubject(userId)
            .claim("role", role)
            .setExpiration(new Date(System.currentTimeMillis() + EXPIRE))
            .signWith(SignatureAlgorithm.HS256, SECRET)
            .compact();
    }

    public static Claims parseToken(String token) {
        return Jwts.parser().setSigningKey(SECRET).parseClaimsJws(token).getBody();
    }
}
