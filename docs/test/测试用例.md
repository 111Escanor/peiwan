# 测试用例

## 一、用户模块

### TC-001 注册普通玩家
- 操作: POST `/api/user/register`，传 `{"username":"test","password":"123","phone":"13800000001","role":"player"}`
- 预期: 返回 code=200，注册成功

### TC-002 注册重复用户名
- 操作: 用已存在的用户名再次注册
- 预期: 返回 code=400，"用户名已存在"

### TC-003 登录成功
- 操作: POST `/api/user/login`，传正确用户名密码
- 预期: 返回 code=200 和 JWT token

### TC-004 登录失败
- 操作: 传错误密码
- 预期: 返回 code=401

### TC-005 登录被禁用账号
- 操作: 管理员禁用某用户后，该用户尝试登录
- 预期: 返回 code=403，"账号已被禁用"

## 二、陪玩师模块

### TC-006 陪玩师列表
- 操作: GET `/api/master/list`
- 预期: 只返回审核通过的陪玩师

### TC-007 陪玩师详情
- 操作: GET `/api/master/detail/{userId}`
- 预期: 返回该陪玩师的完整信息

### TC-008 陪玩师注册
- 操作: 注册时 role 选 master，填写段位、价格、标签
- 预期: 注册成功，陪玩师写入数据库且状态为 pending

## 三、订单模块

### TC-009 创建订单
- 操作: POST `/api/order/create`，传 masterId 和 amount
- 预期: 订单创建成功，status="pending"

### TC-010 模拟支付
- 操作: POST `/api/pay/mock`，传 orderId
- 预期: 支付成功，生成交易流水号，status→"paid"

### TC-011 陪玩师接单
- 操作: POST `/api/order/accept/{orderId}`
- 预期: 接单成功，status→"in_service"

### TC-012 完成服务
- 操作: POST `/api/order/complete/{orderId}`
- 预期: status→"completed"，记录完成时间

### TC-013 取消订单
- 操作: POST `/api/order/cancel/{orderId}`
- 预期: status→"cancelled"

## 四、评价模块

### TC-014 提交评价
- 操作: POST `/api/review/create`，传 orderId、rating、comment
- 预期: 评价成功

### TC-015 查询评价
- 操作: GET `/api/review/order/{orderId}`
- 预期: 返回该订单的评价

## 五、管理后台

### TC-016 数据统计
- 操作: GET `/api/admin/statistics`
- 预期: 返回用户数、订单数、待审核陪玩师数

### TC-017 陪玩师审核
- 操作: POST `/api/admin/approve-master/{userId}?approved=true`
- 预期: 审核通过，该陪玩师出现在列表中

### TC-018 用户禁用/启用
- 操作: POST `/api/admin/toggle-user/{userId}`
- 预期: 用户 status 在 1/0 间切换
